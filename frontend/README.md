# Frontend (Vue 3 + Vite)

개발자가 빠르게 전반을 파악할 수 있도록 개념과 책임 위주로 요약했습니다.

## 전체 구성
- Vue 3 + TypeScript, Vite 번들
- UI: Vuetify
- 상태관리: Pinia
- 라우터: Vue Router (history)
- 통신: Axios 기반 공용 클라이언트

## Directory Structure
- `src/main.ts`, `src/App.vue`: 앱 부트스트랩과 전역 컨테이너.
- `src/layouts/`: 공개/인증 구역을 나누는 레이아웃.
- `src/routes/`: 라우트 선언과 전역 가드. 모듈별(auth/app/legal/info/system)로 나뉘며, 접근 제어는 라우트 메타와 JWT 상태로 판단.
- `src/stores/`: Pinia 스토어. 인증 토큰/사용자 정보를 관리.
- `src/api/` + `src/services/`: API 호출 로직과 서비스 계층을 분리하여 관리.
- `src/views/`: 화면 단위 컴포넌트. 공개(auth) 흐름과 앱 내부(Home/Market/Alerts/Settings/Profile), 시스템(NotFound)로 구분.
- `src/components/`: 재사용 UI 조각(예: 인증 카드 셸).
- `src/utils/`: JWT 등 공용 유틸.
- `src/styles/`: 전역 스타일.

## routing
- Public 영역(`/auth`, `/legal`, `/info`)은 공개 레이아웃을 사용.
- App 영역(`/`)은 인증 필요 레이아웃을 사용하며 기본적으로 로그인·이메일 등록·인증 상태를 요구.
- 전역 가드가 JWT를 해석해 guest / unenrolled / unverified / verified 상태로 분류하고, 라우트 메타의 `allows`와 매칭되지 않으면 적절한 화면으로 리다이렉트.

## 상태 관리 개념
- 인증 스토어: 액세스 토큰을 영속(로컬스토리지) 및 메모리에 보관하고 로그인·가입·재발급·로그아웃 등 흐름을 수행.
- 사용자 스토어: `/user/me` 기반의 프로필 정보를 보관/초기화.

## API 클라이언트 개념
- 공용 Axios 인스턴스가 기본 URL과 쿠키 동반 설정을 포함하고, 요청 시 토큰을 헤더로 붙입니다.
- 401(토큰 문제) 응답 시 재발급을 시도하고 대기 중 요청을 큐잉 후 재시도합니다.
- 서비스 계층은 엔드포인트 래퍼를 감싸 성공/실패만 단순화해 스토어와 뷰에서 쓰기 쉽게 제공합니다.

## 인증 흐름 개념
1) 로그인/가입/재발급 후 토큰을 저장.
2) 라우터 가드가 토큰의 만료 여부와 이메일 등록·인증 플래그를 검사해 접근을 허가/차단.
3) API 호출 중 토큰 만료 시 클라이언트가 재발급을 시도하고, 실패하면 로그인 흐름으로 전파.
4) 로그아웃은 서버와 무관하게 로컬 토큰을 정리해 즉시 세션을 종료.

## 화면 그룹
- 공개: 로그인, 회원가입(선택·약관·이메일), 이메일 인증, 비밀번호 찾기/재설정, OAuth 실패 안내.
- 앱: 홈, 마켓, 알림, 설정, 프로필(현재는 플레이스홀더 중심).
- 시스템: 404 NotFound.
